## Component

`Component` 는 스프링 컨테이너가 객체를 관리하게끔 지정해주는 annotation입니다. Bean에 등록되면서 동작하게 되는데 자세한 내용은 나중에 다루도록 하겠습니다. 아래와 같이 `Component` 을 붙여주면 관리를 할 수 있게 됩니다.

```java
@Component
public class UrlEncoder implements IEncoder

@Component
public class Base64Encoder implements IEncoder
```

## IoC 예제

그럼 스프링에서 관리되는 객체는 어떻게 접근해서 사용하는지에 대한 의문이 생기게 됩니다. ApplicationContextProvider란 클래스를 하나 만들어 스프링 주입되는 context를 받아옵니다.

```java
package com.example.springioc;

import org.springframework.beans.BeansException;
import org.springframework.context.ApplicationContext;
import org.springframework.context.ApplicationContextAware;
import org.springframework.stereotype.Component;

@Component
public class ApplicationContextProvider implements ApplicationContextAware {

    private static ApplicationContext context;

    @Override
    public void setApplicationContext(ApplicationContext applicationContext) throws BeansException {
        context = applicationContext;
    }

    public static ApplicationContext getContext(){
        return context;
    }
}
```

객체를 받아오게되면 `@Component`를 붙힌 Base64와 Url 인코더를 불러올 수 있습니다.

```java
	public static void main(String[] args) {

		SpringApplication.run(SpringIocApplication.class, args);
		// Spring에서 관리하는 컨텍스트 가져옴
		ApplicationContext context = ApplicationContextProvider.getContext();

		// Base64와 Url을 가져옴 (class 이름으로)
		Base64Encoder base64Encoder = context.getBean(Base64Encoder.class);
		UrlEncoder urlEncoder = context.getBean(UrlEncoder.class);

		Encoder encoder = new Encoder(base64Encoder);
		String url = "www.naver.com/books/it?page=10&size=20&name=spring-boot";

		String result = encoder.encode(url);
		System.out.println(result);

		encoder.setIEncoder(urlEncoder);
		result = encoder.encode(url);
		System.out.println(result);
	}

// 결과: 잘나오는 것을 확인할 수 있다.

```

## Qualifier

그렇다면 Encoder 객체를 Bean으로 관리할 수 없을까? 라는 의문이 생깁니다. 물론 할 수 있지만 현재 상태에선 Component로 등록된 객체가 2개 이상이므로 에러가 날 것입니다. 이때 어떤 Bean을 쓸 것인지를 지정해주는 것이 `@Qulifier`입니다.

```java
@Component
public class Encoder {
    private IEncoder iEncoder;

		// base64란 인코더를 들고올꺼라고 지정해줍니다.
		// 보통 해당 클래스의 소문자형으로 이름이 정해집니다.
    public Encoder(@Qualifier("base64Encoder") IEncoder iEncoder){
        this.iEncoder = iEncoder;
    }
	...
}
```

이렇게 Encoder까지 등록하면 아래와 같이 개발자가 new로 객체를 만들어 사용하지 않고 모든 권한이 스프링에게 넘어가게됩니다. 이를 제어가 역전이 됐다해 IoC라 합니다.

```java
public static void main(String[] args) {

		SpringApplication.run(SpringIocApplication.class, args);
		ApplicationContext context = ApplicationContextProvider.getContext();

		// 더이상 따로 받을 필요가 사라졌다.
		//Base64Encoder base64Encoder = context.getBean(Base64Encoder.class);
		//UrlEncoder urlEncoder = context.getBean(UrlEncoder.class);

		// Encoder로 한번에 컨텍스트를 받아온다.
		Encoder encoder = context.getBean(Encoder.class);
		String url = "www.naver.com/books/it?page=10&size=20&name=spring-boot";
		String result = encoder.encode(url);
		System.out.println(result);
	}
```

## Configuration

여태까지 하나의 클래스에서 하나의 Bean을 등록했었는데 두개 이상의 빈을 등록하고 싶으면 `@Configuration` annotation을 사용합니다.  Component annotation이랑 비슷한데 이는 `@Configuration`안에도 Component annotation을 가지고 있기 때문입니다.

Encoder 클래스에서 Component를 해제해주고 Main에다가 AppConfig라는 클래스를 하나 생성해줍니다. Encoder를 불러와 `@Bean` 을 붙혀 직접 등록시켜줄 수 있습니다. 

여기서 base64Encoder와 urlEncoder 이름은 이미 존재하기 때문에 충돌할 위험이 있기에 다른 이름으로 지정해줍니다.

```java
@Configuration
class AppConfig{

	@Bean("base64Encode")
	public Encoder encoder(Base64Encoder base64Encoder){
		return new Encoder(base64Encoder);
	}

	@Bean("urlEncode")
	public Encoder encoder(UrlEncoder urlEncoder){
		return new Encoder(urlEncoder);
	}
}
```

Bean을 가져오는 type에는 이름하고 클래스로 찾을 수 있는데 이번에는 이름을 추가해 해당 맞는 이름과 같은 빈을 불러올 수 있습니다.

```java
Encoder encoder = context.getBean("urlEncode",Encoder.class);
```
