## 선언위치에 따른 변수의 종류

```java
class Variables{
		int iv;           // 인스턴스 변수
		static int cv;    // 클래스변수(static변수, 공유변수)
		
		void method()
		{
			int iv = 0; // 지연 변수
		}
}
```

### 1. 인스턴스 변수(instance variable)

- 클래스 영역에 선언된다.
- 클래스의 인스턴스를 생성할 때 만들어진다.
- 인스턴스마다 고유한 상태를 유지해야하는 속성일 경우, 인스턴스 변수로 선언한다.

### 2. 클래스 변수(class variable)

- 클래스 영억에 선언된다.
- 인스턴스 변수 앞에 static을 붙이기만 하면 된다.
- 모든 인스턴스가 공통된 저장공간을 공유한다.
- 인스턴스를 생성하지 않고 바로 사용가능하다. (예: Variables.cv)
- 메모리에 로딩(loading)될 때 생성되어 프로그램이 종료될 때까지 유지된다.
- public을 앞에 붙이면 전역변수(global variable)의 성격을 갖는다.

### 3. 지역 변수 (local variable)

- 메서드 내에 선언되어 메서드 안에서만 사용가능하다.
- 메서드가 종료되면 소멸되어 다른 곳에서 사용이 불가능하다.

## 클래스변수와 인스턴스변수의 차이점

> 인스턴스 변수는 인스턴스가 생성될 때마다 생성되므로 인스턴스마다 각기 다른 값을 유지할 수 있다.
클래스 변수는 모든 인스턴스가 하나의 저장공간을 공유하므로 항상 공통된 값을 갖는다.
> 

카드 게임을 예로 들자면 카드의 속성에는 카드 무늬, 숫자, 폭, 높이가 있다고 가정합니다. 이를 토대로 클래스를 작성하게 되면 아래와 같습니다.

```java
class Card{
		String kind; // 무늬
		int number;  // 숫자

		static int width = 100;  // 폭
		static int height = 250; // 높이
}
```

카드마다 고유한 무늬와 숫자를 지니는데 이는 인스턴스 마다 고유한 값이라고 할 수 있습니다. 따라서 무늬와 숫자는 인스턴스 변수로 선언해줄 수 있습니다. 하지만 모든 카드의 폭과 높이는 같아야하므로 이는 클래스변수로 선언해줘 어떠한 카드든 폭과 높이는 같게 세팅해줄 수 있습니다.

<aside>
💡 여기서 인스턴스 c1과 c2를 만들어 c1.width 같이 클래스변수를 수정해줄 수 있습니다.
하지만 이는 인스턴스 변수라고 오해할 수 있어 클래스변수를 수정할 땐 Card.width와 같이 수정하는 것을 권장합니다.

</aside>
