## Singleton pattern

- Singleton 패턴은 어떠한 클래스(객체)가 유일하게 1개만 존재할 때 사용한다.
- 주로 서로 자원을 공유할 때 사용하는데, 실물 세계에서는 프린터가 해당되며, 실제 프로그래밍에서는 TCP Socket 통신에서 서버와 연결된 connect 객체에 주로 사용된다.

---

## Singleton 예제

```java
package com.company.singleton;

public class SocketClient {
    private static SocketClient socketClient = null;

    // 외부에서 생성자로 인스턴스를 생성하지 못하게 private로 막아놓음
    private SocketClient(){

    }

    // 초기 한번만 소켓 클라이언트를 생성함
    public static SocketClient getInstance(){
        if(socketClient == null){
            socketClient = new SocketClient();
        }
        return socketClient;
    }
    
    public void connect(){
        System.out.println("connect");
    }
}
```

- 단 하나의 소켓 클라이언트를 위해 private로 접근하지 못하게 만들어준다.
- getInstance란 메소드를 통해 초기에 단 한번만 객체의 인스턴스를 만들어낸다.

---

## Singleton의 장점

1. 효율적인 메모리 관리
    
    하나의 인스턴스만을 만들기 때문에 고정된 메모리 영역만을 사용한다.
    
    이미 생성된 인스턴스를 사용하니 속도도 조금 더 빠르다.
    
2. 데이터 공유
    
    전역으로 사용되는 인스턴스이기에 다른 클래스의 인스턴스들이 접근하기 용이하다.
    
    동시성 문제만 해결되면 좋다.
    

---

## Singleton의 문제점

그렇다고 마냥 Singleton이 장점만 있는 것은 아니다.

1. 싱글톤 패턴에 많은 코드가 소요된다.
    
    동시성 문제를 해결해야하기 때문이다.
    
2. 테스트하기 힘들다.
    
    싱글톤의 인스턴스는 전역에 공유되어 있기 때문에 테스트를 하려면
    
    초기화를 하고 다시 테스트해야하는 불편함이 존재한다.
    
3. DIP를 위배한다.
    
    클라이언트가 구체 클래스에 의존하는 현상이 발생한다.
    
    또한 높은 확률로 OCP도 위배할 가능성이 있다.
