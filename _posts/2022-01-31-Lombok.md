## Lombok

![](https://jihun-24k.github.io/assets/images/spring/lombok1.png){: .align-center}

자바에서 Model Object를 만들 때, 맴버 필드에 대한 Getter/Setter, ToString과 맴버필드에 주입하는 생성자를 만드는 코드 등 불 필요하게 반복적으로 만드는 코드를 어노테이션을 통해 줄여 주는 라이브러리입니다.

---

### Lombok을 사용하는 이유

- 어노테이션 기반의 코드 자동 생성을 통한 생산성 향상
- 반복되는 코드 다이어트를 통한 가독성 및 유지 보수성 향상
- Getter, Setter 외에 Builder 패턴이나 로그 생성등 다양한 방면으로 활용 가능

---

## Lombok Annotation 종류

### Getter/ Setter

**Getter**: 객체 외부에서 객체 필드 값을 사용하기 부적절한 경우, 메소드로 필드 값을 가공 후, 외부로 전달해줍니다.

**Setter**: 데이터는 외부에서 접근하지 않도록 막고, 메소드는 공개하여 외부에서 메소드를 통해 데이터에 접근하도록 유도해줍니다.

`적용 방법`

- 클래스 위에 적용시키면 모든 변수들에 적용
- 변수 이름 위에 적용시키면 해당 변수들만 적용

### Data

Getter와 Setter를 포함한 toString 메소드까지 함께 만들어주는 어노테이션입니다.

### AllArgsConstructor

모든 변수를 사용하는 생성자를 자동 완성 시켜주는 어노테이션입니다. 

### NoArgsConstructor

어떠한 변수도 사용하지 않는 기본 생성자를 자동완성 시켜주는 어노테이션입니다.

```java
@Data
@NoArgsConstructor
@AllArgsConstructor
public class User {
    private String name;
    private int age;
}
```

### Builder

해당 클래스의 객체의 생성에 Builder 패턴을 적용시켜줍니다. 

```java
Restaurant restaurant = restaurantService.addRestaurant(
                Restaurant.builder()
                .name(resource.getName())
                .address(resource.getAddress())
                .build()); 
```

builder을 사용하면 코드의 가독성이 굉장히 좋아지는 것을 확인할 수 있다.

### Slf4j

System.out으로 log를 찍지않고 log란 명령어로 찍게 만들어주는 어노테이션입니다.

```java
@Slf4j
@RestController
@RequestMapping("/api/user")
public class ApiController {
    @PostMapping("")
    public User user(@RequestBody User user){
        log.info("User : {}",user);
        return user;
    }
}
```
