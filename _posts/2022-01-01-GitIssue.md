## Issue란?

프로젝트에서 나온 모든 일(각종 버그, 풀 리퀘스트 등등)을 이슈라고 말한다.

## Git Issue

![](https://jihun-24k.github.io/assets/images/git/gitissue1.png){: .align-center}

Git Issue는 깃 공식문서에 나와있듯이 깃헙에서 나온 이슈들을 관리하기 위한 기능입니다. 

### Git Issue 생성

어떠한 git repository에서 이슈탭에 들어가면 아래와 같은 화면이 나오는데 여기서 `New issue` 버튼을 눌려줍니다.
![](https://jihun-24k.github.io/assets/images/git/gitissue2.png){: .align-center}

기본 이슈 생성페이지는 아래와 같고 타이틀과 내용을 작성할 수 있는 페이지가 나옵니다. 이슈화할 내용을 적어놓고 생성을 할 수 있습니다.
![](https://jihun-24k.github.io/assets/images/git/gitissue3.png){: .align-center}

### assignees & Label 

위와 같이 이슈들을 관리하다가 이슈들이 많아지게되면 자기가 봐야하는 이슈들을 모를 수도 있을 것입니다. 이 이슈의 책임자 또는 관련된 사람이 누구인지 또는 무슨 카테고리에 있는 이슈인지 알면 더 직관적이고 효율적으로 볼 수 있을 것입니다. 여기서 책임자나 관련된 사람이 누구인지 나타내는 기능이 `assigness`입니다.

Issue를 생성하는 오른쪽 부분에 `Assingees`란 부분에서 사람을 지정할 수 있습니다. (현재 레파지토리는 개인 레파지토리로 당사자밖에 없다...)

![](https://jihun-24k.github.io/assets/images/git/gitissue5.png){: .align-center}

또한 Label이란 기능을 통해 해당 이슈가 어떤 카테고리에 속하고 어떤 이슈인지 더 명확하게 설명이 가능합니다. 예를들어 버그이면 버그라고 라벨을 설정하게되면 이 이슈는 버그에 관련된 이슈구나를 바로 파악할 수 있게 됩니다.

Label 또한 Assignees과 같이 이슈를 생성하는 오른쪽 부분에 있습니다. 자기가 원하는 라벨을 만들어서 지정할 수도 있고 원래 있던 라벨을 지정할 수 있습니다. 아래와 같이 새로운 라벨에 대해 생성을 하고 적용시킬 수 있습니다.

![](https://jihun-24k.github.io/assets/images/git/gitissue6.png){: .align-center}
