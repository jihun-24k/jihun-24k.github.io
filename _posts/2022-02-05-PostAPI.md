## POST API

### POST API(Map)

GET API를 작성할 때와 마찬가지로 Map으로 먼저 받아오는 코드를 작성해보았습니다. 여기서 GET과 다른점은 `@RequestBody` annotation을 사용한다는 점입니다.

```java
@RestController
@RequestMapping("/api")
public class PostApiController {

    @PostMapping("/post")
    public void post(@RequestBody Map<String, Object> requestData){
        requestData.forEach((key, value) ->
        {
            System.out.println("key: "+key);
            System.out.println("value: "+value);
        });
    }
}
```

### POST API(dto)

POST도 마찬가지로 받아오는 데이터의 값을 미리 dto로 지정해줄 수 있습니다. GetApi때와 같이 dto를 하나 만들어주고 그 안에 들어오는 `key값` 들로 설정해준 다음 키 값들에 맞는 value들을 들고 올 수 있게 해줍니다.

Controller에선 객체를 생성해준다음 사용하면 아래와 같습니다.

```java
@RestController
@RequestMapping("/api")
public class PostApiController {

    @PostMapping("/post")
    public void post(@RequestBody PostRequestDto requestData){
        System.out.println(requestData);
    }
}

//결과
//PostRequestDto{account='user', email='steve@gamil.com', address='부산', password='abcd'}
```

### JsonProperty

하지만 Spring은 camelCase 사용하는데 JSON의 키값이 snake_case나 다른 케이스를 사용한다면 키값을 읽어오지 못하는 경우가 발생합니다. 이 경우에는 많은 해결책들이 있지만 해당 변수에 `@JsonProperty` annotation을 사용해주면 해결이 됩니다. (추후에 `@JsonNaming`을 다뤄볼 예정)

```java
		private String phoneNumber;

    public String getPhoneNumber() {
        return phoneNumber;
    }

    public void setPhoneNumber(String phoneNumber) {
        this.phoneNumber = phoneNumber;
    }

//어노테이션 쓰기 전결과
//PostRequestDto{account='user', email='steve@gamil.com', address='부산', password='abcd', phoneNumber='null'}
```

```java
@JsonProperty("phone_number") //json에서의 키이름을 적어준다.
private String phoneNumber;

//결과
//PostRequestDto{account='user', email='steve@gamil.com', address='부산', password='abcd', phoneNumber='010-2222-3333'}
```
