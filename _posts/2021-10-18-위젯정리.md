### 위젯이란?

플러터에서 위젯이란 화면에 표시되는 모든 것, UI 컴포넌트라고도 합니다. 예를 들어 안드로이드에서는 앱 화면에 표시되는 것들을 View 클래스로 정의하는데 이를 플러터에서는 위젯이라 함은 이해하기 쉬울 거 같습니다.

플러터의 위젯은 크게 <b>Stateless 위젯</b>과 <b>Stateful 위젯</b>으로 구별이 됩니다.

---

### Stateless 위젯

Stateless 위젯은 화면 표시용 위젯입니다. 위젯이 로딩돼 화면에 표시된 이후에는 그 위에 이벤트나 동작을 해도 내용을 변경할 수 없습니다. Stateless 위젯은 <b>StatelessWidget</b> 클래스입니다.

```dart

import 'package:flutter/material.dart';

void main() => runApp(MaterialApp(
  title: 'Stateless Widget Demo',
  home: Scaffold(
    appBar: AppBar(title: Text('Stateless 위젯 데모')),
    body: _FirstStatelessWidget(),
  ),
));

class _FirstStatelessWidget extends StatelessWidget {
  @override
  Widget build(BuildContext context){
    return Text('이것은 stateless 위젯입니다.');
  }
}
```

 Stateless 위젯을 만들기 위해서는 먼저 StatelessWidget 클래스를 상속하고 build() 메서드에서 내가 원하는 위젯을 반환하면 됩니다.

---

### Stateful 위젯

 Stateful 위젯은 Stateless 위젯과 다르게 build() 메서드가 없습니다. 대신 클래스를 두개로 분할해줍니다. Stateful 위젯은 버튼을 누르거나 터치를 했을 때 화면에 변경된 내용을 표시할 수 있기에 최대한 변동부(변화가 있는 부분)과 고정부(변화가 없는 부분)을 나누어 설계하기 위함입니다.

```dart
import 'package:flutter/material.dart';

void main() => runApp(MaterialApp(
  title: 'Stateless -> Stateful 위젯 데모',
  home: Scaffold(
    appBar: AppBar(title: Text('Stateless -> Stateful 위젯 데모')),
    body: _FirstStatefulWidget(),
  ),
));

class _FirstStatefulWidget extends StatefulWidget{
  @override
  State createState() => _FirstStatefulWidgetState();
}

class _FirstStatefulWidgetState extends State<_FirstStatefulWidget>{
  @override
  Widget build(BuildContext context){
    return Text('이것은 stateful 위젯입니다.');
  }
}
```

위의 코드에선 고정부는 _FristStatefulWidget 클래스이고 변동부는 _FirstStatefulWidgetState 클래스입니다.

### Stateful 위젯 생명주기

#### 생명주기란?
위젯의 생명주기란 일종의 앱이 실행되는 순서라고 이해하면 편합니다. 언제 데이터를 주고 받을지, 화면이 바뀔 때 어떤 로직을 수행하는지를 명확하게 설계하기 위해 생명주기를 지원합니다.


| 순서 | 메서드 | 비고 |
|:-----:||:-----:||:-----:|
| 1 | StatefulWidget.createState() | 상태를 생성하며 해당 메서드를 제외하곤 나머진 State 클래스에 있다 |
| 2 | mounted 변수가 true가 된다 | 화면에 위젯이 부착된 상태|
| 3 | <b>initState()</b> | 상태를 초기화할 수 있고 단 한번만 호출이 가능하다 |
| 4 | didChangeDependencies() | 상태 객체의 의존성이 부착된 상태 |
| 5 | <b>build()</b> | 위젯을 화면에 표시하는 메서드 |
| 6 | didUpdateWidget() | 위젯의 설정이 변경될 때 호출 |
| 7 | <b>setState()</b> | 위젯의 상태를 갱신, 위젯을 처음부터 만들지만 initState()는 호출이 안된다 |
| 8 | deactivate() | 상태 객체가 위젯 트리에서 제거됨, 경우에 따라 다시 위젯 트리에 추가될 수 있다 |
| 9 | dispose() | 상태 객체가 위젯 트리에서 완전히 제거됨, 이 메서드가 호출되면 상대 객체는 더 이상 재사용할 수 없다 |
| 10 | mounted가 false로 설정됨 | 최종적으로 위젯이 화면에서 탈착됨 |

#### 간단한 UI를 추가한 예제 코드

```dart
import 'package:flutter/material.dart';

void main() => runApp(MaterialApp(
  title: 'Stateful 위젯 데모',
  home: Scaffold(
    appBar: AppBar(title: Text('Stateful 위젯 데모')),
    body: _MyStatefulWidget(),
  ),
));

class _MyStatefulWidget extends StatefulWidget{
  @override
  State createState() => _MyStatefulWidgetState();
}

class _MyStatefulWidgetState extends State<_MyStatefulWidget>{
  String _buttonState;

  @override
  void initState() {
    super.initState();
    print('initState(): 기본값을 설정합니다.');
    _buttonState = "OFF";
  }

  @override
  void didChangeDependencies() {
    print('didChangeDependencies() 호출됨');
  }

  @override
  Widget build(BuildContext context) {
    print('build() 호출됨');
    return Column(
      children: <Widget>[
        RaisedButton(
          child: Text('버튼을 누르시오.'),
          onPressed: _onClick,
        ),
        Row(
          children: <Widget>[
            Text('버튼 상태: '),
            Text(_buttonState),
          ],
        ),
      ],
    );
  }

  void _onClick() {
    print('_onClick() 호출됨');
    setState(() {
      print('setState() 호출됨');
      if (_buttonState == 'OFF'){
        _buttonState = 'ON';
      }else{
        _buttonState = 'OFF';
      }
    });
  }

  @override
  void didUpdateWidget(_MyStatefulWidget oldWidget) {
    print('didUpdateWidget() 호출됨');
  }

  @override
  void deactivate() {
    print('deactivate()');
  }

  @override
  void dispose() {
    print('dispose()');
  }
}
```
