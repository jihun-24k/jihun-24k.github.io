## Strategy pattern

전략 패턴으로 불리며, 객체지향의 꽃이다.

유사한 행위들을 캡슐화하여, 객체의 행위를 바꾸고 싶은 경우 직접 변경하는 것이 아닌 전략만 변경하여, 유연하게 확장하는 패턴

SOLID 중에서 OCP와 DIP를 따른다.

---

## Strategy pattern에서 필요한 3가지

- 전략 메서드를 가진 전략 객체 (Normal Strategy, Base64 Strategy)
- 전략 객체를 사용하는 컨텍스트 (Encoder)
- 전략 객체를 생성해 컨텍스트에 주입하는 클라이언트 (Main method)

---

## Strategy pattern 예제

### Encoder (뼈대 클래스)

```java
package com.company.strategy;

public class Encoder {

    private EncodingStrategy encodingStrategy;

    public String getMessage(String message){
        return this.encodingStrategy.encode(message);
    }

    public void setEncodingStrategy(EncodingStrategy encodingStrategy){
        this.encodingStrategy = encodingStrategy;
    }
}
```

### NormalStrategy (전략 객체)

```java
package com.company.strategy;

public class NormalStrategy implements EncodingStrategy{
    @Override
    public String encode(String text) {
        return text;
    }
}
```

### Base64Strategy (전략 객체)

```java
package com.company.strategy;

import java.util.Base64;

public class Base64Strategy implements EncodingStrategy{
    @Override
    public String encode(String text) {
        return Base64.getEncoder().encodeToString(text.getBytes());
    }
}
```

### Main (전략 객체 생성 및 주입하는 클라이언트)

```java
package com.company;

import com.company.strategy.*;

public class Main {

    public static void main(String[] args) {
        Encoder encoder = new Encoder();

        //base64
        EncodingStrategy base64 = new Base64Strategy();

        //normal
        EncodingStrategy normal = new NormalStrategy();

        String message = "hello java";

        encoder.setEncodingStrategy(base64);
        String base64Result = encoder.getMessage(message);
        System.out.println(base64Result);

        encoder.setEncodingStrategy(normal);
        String normalResult = encoder.getMessage(message);
        System.out.println(normalResult);
    }
}
```

- Encoder라는 뼈대가 되는 클래스는 변경하지 않고 Normal이나 Base64등과 같은 전략 객체들을 만들어 상황에 따라 바꿔준다
