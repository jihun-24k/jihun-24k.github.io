### 플러터 설치 과정

이번에 플러터 스터디를 진행하게 되면서 플러터 개발환경을 갖춰야했습니다. 워낙 구글에서 지원을 잘해놓아서 공식 문서에 들어가 설치 홈페이지를 보면 윈도우, 맥, 리눅스등 각 OS 별로 설치과정을 잘 설명 되어있었습니다. 

이 게시물에선 맥OS 환경에서 설치하는 과정을 설명해볼 생각입니다. 플러터를 시작하기 전에 아래 세 가지를 설정해야하는데 구글에서 추천하는 IDE가 안드로이드 스튜디오라 IDE는 안드로이드 스튜디오로 진행하기로 했습니다.

1. 플러터SDK: 플러터 개발을 위함 개발자 킷
2. 안드로이드 스튜디오: 안드로이드 개발을 위한 IDE
3. 안드로이드SDK: 안드로이드 개발을 위한 SDK

### 플러터 SDK 다운로드

![](https://jihun-24k.github.io/assets/images/flutter/install1.png){: .align-center}

압축된 flutter sdk를 원하는 위치에 압축을 풀어라고 하지만 필자는 바로 Downloads에 압축이 풀린 상태로 저장이 돼 다른 파일에 옮겨 놓았습니다.

```bash
$ cd ~/development
$ unzip ~/Downloads/flutter_macos_vX.X.X-stable.zip
```

플러터 도구를 환경변수 path에다가 추가합니다.

```bash
$ export PATH="$PATH:`pwd`/flutter/bin"
```

그 다음으론 공식 문서에서 선택적으로 플러터 개별 바이너리를 사전 다운로드해라고 했는데 아래의 명령어로 설치할 수 있습니다.

```bash
$ flutter precache
```

<b>flutter doctor</b>는 현재 환경에서 설정을 완료하는 데 필요한 플랫폼 의존성이 있는지 확인하기 위해서 사용됩니다.

```bash
$ flutter doctor
```

### 플랫폼 설정

플러터는 안드로이드와 iOS 둘 다 개발이 가능한 크로스 플랫폼으로 이 게시글에선 공식 문서에서처럼 두 가지 환경을 다 설정합니다.

---

### iOS 설정

#### Xcode 설치
  iOS용 플러터 앱 개발을 위해선 Xcode가 설치된 맥이 필요합니다. 먼저 웹이나 애플 스토어에서 최신 안정 버전의 <b>Xcode</b>을 다운 받습니다.
  다운이 다 됐으면 아래의 명령어를 실행해 Xcode 커맨드라인 도구를 설정해줍니다.
  ```bash
  $ sudo xcode-select --switch /Applications/Xcode.app/Contents/Developer
  $ sudo xcodebuild -runFirstLaunch
  ```

#### 시뮬레이터 설정
  플러터로 개발할 때 사용되는 앱뮬레이터가 잘 열리는지 확인해봅니다. 아래의 명령어를 입력하고 화면이 잘 나오는지 확인합니다.
  ```bash
  $ open -a Simulator
  ```
  ![](https://jihun-24k.github.io/assets/images/flutter/install2.png){: .align-center}{: width="40%" height="40%"}
  
  위의 사진처럼 시뮬레이터가 나온다면 잘 설정된 것입니다.

### 안드로이드 설정

안드로이드는 IDE로 안드로이드 스튜디오를 쓸거기 때문에 공식페이지에 가서 설치해줍니다. 안드로이드 IDE를 설치하고 바로 SDK를 설치해야합니다. 위의 플러터 SDK와 같은 홈디렉토리 안에 설치하는게 개발할 때 편하기 때문에 경로를 커스텀해 설치해줍니다. 마지막으로 ANDRPID_HOME 환경 변수를 설정하고 flutter doctor를 실행하여 안드로이드 SDK 설치를 확인합니다.
