## Filter

웹 어플리케이션에서 관리되는 영역으로써 Spring Boot Framework에서 Client로 부터 오는 요청/응답에 대해 최초/최종 단계의 위치에 존재하며, 이를 통해서 요청/응답의 정보를 변경하거나 Spring에 의해서 데이터가 변환되기 전의 순수한 Client의 요청,응답 값을 확인할 수 있습니다.

ServletRequest, ServletResponse 의 객체를 변환할 수 있습니다.

주로 Spring Framework에서는 request/response의 Logging 용도로 활용하거나 인증과 관련된 Logic들을 해당 Filter에서 처리합니다.

이를 선/후 처리함으로써 Service business logic과 분리 시켜줍니다.

### BufferedReader

Java는 커서로 이동하며 파일을 읽습니다. 한번 파일을 끝까지 다읽으면 다음 객체가 파일을 읽으려고 할 때 에러가 납니다.

GlobalFilter에서 이미 json 파일을 읽어 User 객체를 넘겨줄 때 에러가 뜹니다.

```java
2022-01-25 10:39:46.307  INFO 34209 --- [nio-8080-exec-1] com.example.filter.filter.GlobalFilter   : url : /api/user , line : {
2022-01-25 10:39:46.308  INFO 34209 --- [nio-8080-exec-1] com.example.filter.filter.GlobalFilter   : url : /api/user , line :   "name" : "jihun",
2022-01-25 10:39:46.308  INFO 34209 --- [nio-8080-exec-1] com.example.filter.filter.GlobalFilter   : url : /api/user , line :   "age" : 0
2022-01-25 10:39:46.308  INFO 34209 --- [nio-8080-exec-1] com.example.filter.filter.GlobalFilter   : url : /api/user , line : }
```
