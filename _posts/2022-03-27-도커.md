## Container 정의

- 격리된 공간에서 프로세스가 동작하는 기술

## Docker 정의

- 컨테이너 기반의 오픈소스 가상화 플랫폼

### Environment Disparity

- 개발환경이 서로 맞지 않는 상태를 의미한다.
- 예를 들어 현재 윈도우에서 개발을 하고 서버에 적용시켰는데 서버는 리눅스라 코드가 잘 동작하지 않을 상황이 있다.

## Docker를 사용하는 이유

### Environment Disparity 문제 해결

- 서버와 윈도우에 모두 도커를 설치하고 나서 도커 파일을 생성
- 도커 파일에 자신이 구현하고 싶은 환경을 설정해서 넣는다.
- 도커 파일을 서버와 윈도우에 다 넘겨주게 되면 도커는 해당 파일을 읽고 필요한걸 다운로드 받는다.
- 도커는 해당 설정한 환경과 같은 버츄얼 컨테이너를 컴퓨터에 생성

### 컨테이너의 독립성

- 한 개의 서버에 각기 다른 컨테이너들을 넣는다.
- 여러 환경에서 개발하고 서버를 관리할 때 편해진다.

### 요약

1. 원하는 개발 환경을 파일에 저장하면 docker는 이를 원하는 어떤 머신에든 해당환경을 시뮬레이션 해준다.
2. 이러한 환경은 각각 독립적으로 존재하기 때문에 무슨 환경이든 모듈식으로 관리가 용이하다.
