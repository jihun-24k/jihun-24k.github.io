## PUT API(dto)

이번 put api는 CarDto와 PostRequestDto를 만들어 어떤 사용자가 자동차 리스트를 수정하는 상황으로 코딩을 했습니다. 

여기서 중요한 부분은 자동차는 리스트로 받아오기 때문에 List<CarDto>로 선언해주어 받아줘야합니다. 아래의 JSON을 받는다고 가정합니다. 

```json
{
  "name" : "jihun",
  "age" : 20,
  "car_list" : [
    {
      "name" : "BMW",
      "car_number" : "11가 1234"
    },
    {
      "name" : "AUD",
      "car_number" : "22나 5678"
    }
  ]
}
```

그리고 해당 JSON에 맞는 dto를 작성 후 실행시키면 차의 내용이 null이 나오면서 받아오지 못하는 현상이 발생하는데 이 이유는 다음 annotation으로 해결이 가능합니다.

```java
//getter & setter 코드 생략
public class PostRequestDto { 
    private String name;
    private int age;
    private List<CarDto> carList;	//자동차 정보를 리스트로 받아온다.
}
public class CarDto {
    private String name;
    private String carNumber;
}

//결과
//PostRequestDto{name='jihun', age=20, carList= null}
```

### JsonNaming

하지만 위의 방식으로 받아오게되면 차 정보만 null로 받아오는 상황이 발생합니다. 이는 Spring에서 데이터를 읽어올때 camelCase로 읽어오면서 생기는 문제인데 저번 상황에서는 변수 하나하나를 설정해주는 `@JsonProperty` annotation을 사용했지만 이번엔 해당 객체 클래스의 모든 변수들의 읽는 방식을 바꿔주는 annotation인 `@JsonNaming` 을 사용해서 문제를 해결해보았습니다.

```java
//getter & setter 코드 생략
@JsonNaming(value = PropertyNamingStrategy.SnakeCaseStrategy.class)
public class PostRequestDto { 
    private String name;
    private int age;
    private List<CarDto> carList;	//자동차 정보를 리스트로 받아온다.
}
@JsonNaming(value = PropertyNamingStrategy.SnakeCaseStrategy.class)
public class CarDto {
    private String name;
    private String carNumber;
}

//결과
//PostRequestDto{name='jihun', age=20, carList=[CarDto{name='BMW', carNumber='11가 1234'}, CarDto{name='AUD', carNumber='22나 5678'}]}
```

### Response에 대해서

Request에 비해 Response는 매우 간단한 형태입니다. 따로 작성해줄 필요없이 미리 디자인해놓은 데이터 클래스의 규칙에 따라 return해줍니다. DTO에서 지정해준 snake_case와 데이터 종류들이 잘 Response된 것을 확인 할 수 있습니다.
