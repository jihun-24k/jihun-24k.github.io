## 실무 AOP 예제

### Around

Before과 AfterReturning 만으로는 특정 메소드의 로그 기록을 정확히 찍어낼 수 없습니다. 이때 사용되는 Annotation이 Around입니다.

joinPoint에 proceed를 호출하게 되면 해당 실제 메소드를 실행시켜줍니다. 따라서 시간을 잴때 해당 구문 위아래로 실행전, 실행 후에 관한 코드를 작성할 수 있습니다.

이는 만일 AOP가 지원되지않고 코드로 작성하려면 만들어줬던 Get, Post, Delete 메소드에 각각 다 해당 구문들을 추가시켜줘야하고 이는 로직과는 관계없는 코드들이 반복되는걸 알 수 있습니다. 반복되고 겹치는 부분을 AOP로 따로 빼주면서 좀더 유지보수가 편하게 만들어 줄 수 있습니다.

```java
		@Around("cut() && enableTimer()")
    public void around(ProceedingJoinPoint joinPoint) throws Throwable {
        StopWatch stopWatch = new StopWatch();
        stopWatch.start();

        Object result = joinPoint.proceed();

        stopWatch.stop();

        System.out.println("total time: " + stopWatch.getTotalTimeSeconds());
    }
```
