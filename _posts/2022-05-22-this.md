## 생성자에서 다른 생성자 호출

### this()

생성자 간에도 서로 호출이 가능합니다. 단 아래의 두 조건을 만족해야합니다.

> - 생성자의 이름으로 클래스이름 대신 this를 사용한다.
- 한 생성자에서 다른 생성자를 호출할 때는 반드시 첫줄에서만 호출이 가능하다.
> 

```java
Car (String color){
		door = 5;     // 첫번째 줄
		Car(color, "auto", 4); // 에러 1. 생성자의 두번째 줄에서 다른 생성자 호출
}                          // 에러 2. this(color, "auto", 4);로 해야함
```

첫 줄에서만 호출해야하는 이유는 생성자 내에서 초기화 작업도중에 다른 생성자를 호출하게 되면, 호출된 다른 생성자 내에서도 멤버변수들의 값을 초기화 할 것이므로 다른 생성자를 호출하기 이전의 초기화 작업이 무의미해질 수 있기 때문입니다.

### this.

```java
Car (String c, String g, int d){
		color = c;
		gearType = g;
		door = d;
}
```

왼쪽의 코드는 생성자의 매개변수와 인스턴스변수의 이름이 동일하지 않기에 상관없이 초기화가 잘됩니다. 하지만 오른쪽 코드는 생성자의 매개변수와 인스턴스변수의 이름이 동일해서 만일 this를 붙여주지 않으면 `생성자 매개변수=생성자 매개변수` 로 인식하고 말아버립니다. 따라서 this 사용해 인스턴스 변수란 것을 알려줘야하는데 여기서 this가 인스턴스 변수를 참조하는 참조변수라 할 수 있습니다.

this를 쓰는 이유는 보통 인스턴스 변수는 생성자 매개변수로 초기화하게 되는데 이때 이름이 많이 겹칩니다. 따라서 ‘this’를 사용해 구별되도록 하는것입니다.

## 정리

> **this** - 인스턴스 자신을 가리키는 참조변수, 인스턴스의 주소가 저장되어 있다. 모든 인스턴스메서드에 지역  변수로 숨겨진채로 존재한다.

**this(), this(매개변수)** - 생성자, 같은 클래스의 다른 생성자를 호출할 때 사용한다.
