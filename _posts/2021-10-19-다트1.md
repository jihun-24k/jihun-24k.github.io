### 다트 언어란?

![](https://jihun-24k.github.io/assets/images/flutter/dart1.jpg){: .align-center}

다트언어는 웹을 사용하기 위해서 만들어진 언어입니다. 하지만 웹 분야에선 그리 활용되지 못하고 다트2.0이 되면서 클라이언트 언어로서 <b>사운드 타입 시스템</b>과 <b>XML 없이 UI로 코드 구성</b>을 지원합니다.

다트는 정적 타입를 기반으로 var과 dynamic 같은 동적 타입 기능을 제공하며 이를 사운드 타입 시스템이라 부릅니다. 또한 플러터는 UI를 만들 때 별도의 마크업 언어를 사용하지 않습니다. 이러한 장점을 가진 다트로 UI를 구성하면 코드가 서로 얽히지 않고 비교적 깔끔하게 정리할 수 있습니다.

### 주석, 변수, 데이터 타입

| 구분 | 데이터 타입 | 비고 |
|:-----:||:-----:||:-----:|
| 수치형 | int | -2^63 ~ - 2^63-1 |
| | double | IEEE 754 표준을 따름 |
| | num | int와 double의 상위 타입 |
| 부울형 | bool | true/false |
| 문자형 | String | |
| 동적 타입 | var | 변수 할당 시 타입이 지정됨 |
| | dynamic | 타입을 특정하지 않음 |
| 리스트 | List | 다트에 배열은 없음 |
| 집합 | Set | |
| 맵 | Map | |

- var은 진정한 의미에서 동적 타입은 아닙니다. 할당을 통해 타입이 결정되면 그 이후에는 타입을 변경할 수 없습니다.
- dynamic은 타입이 결정되지 않고 계속 변경이 가능합니다.

### 연산자

대표적인 다트 전용 연산자만 표기를 했으며 앱을 만들 때 잘 사용되지 않는 비트 연산자와 시프트 연산자는 생략했습니다.

| 구분 | 연산자 | 비고 |
|:-----:||:-----:||:-----:|
| 할당 연산자 | ??= | 대상이 null인 경우에만 우변의 값을 할당함 |
| 산술 연산자 | ~/ | 몫을 구함(다트 전용) |
| 관계 연산자 | <, >, <=, >=, ==, != | |
| 논리 연산자 | &&, || |
| 삼항 연산자 | ? | |

다트 언어에서 유용하게 쓸 수 있는 응용 연산자 목록은 다음과 같습니다.

| 구분 | 연산자 | 비고 |
|:-----:||:-----:||:-----:|
| 타입 테스트 연산자 | as | 타입을 강제로 형변환 |
| | is | 객체가 특정 타입이면 true |
| | is! | 객체가 특정 타입이면 false |
| if null 연산자 | 표현식1 ?? 표현식2 | 표현식1이 null이 아니면 표현식 1의 값, 만약 null이면 표현식2의 값을 반환함 |
| 케이스 케이드(casecade) 연산자 | _ | |
| 조건적 멤버 접근 연산자 | ?. | 객체가 null이 아닐 때만 멤버의 값을 반환, 만약 nulldlaus null을 반환 |

### final, const, static

현대 프로그래밍에서는 변수 앞에 다양한 키워드를 붙여 용도를 분명이 하고 있습니다. 대표적으로 C언어나 자바가 있습니다. 다트 또한 그 흐름을 따르고 있으며 그 종류는 아래와 같습니다.

1. final
    - 어떤 변수를 참조하는 값이 한번 설정되면 다른 값으로 변경될 수 없다는 의미입니다. 어떤 변수를 선언하고 실행 시간에 한 번만 결정하는 경우 활용할 수 있습니다.
2. const
    - 상수를 만드는 방법입니다. 자바는 상수를 만들 때 public static final이지만 다트에선 const로 단순화가 할 수 있습니다.
3. static
    - 단순히 인스턴스를 만들지 않고 어떤 값을 출력할 수 있습니다.
