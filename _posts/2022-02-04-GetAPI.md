## GetMapping과 RequestMapping의 차이점

```java
package com.example.hello.controller;

import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestMethod;
import org.springframework.web.bind.annotation.RestController;

@RestController
@RequestMapping("/api/get")
public class GetApiController {

    @GetMapping(path = "/hello")
    public String hello(){
        return "get hello";
    }

    @RequestMapping(path = "/hi", method = RequestMethod.GET) 
		// get http://localhost:8080/api/get/hi
    public String hi(){
        return "hi";
    }
}
```

`RequestMapping` 은 get, post, delete, put을 다 지원하는 annotation입니다. 따라서 get방식으로만 지정하고 싶다면 method = RequestMethod.GET이라고 지정해주면 됩니다. 예전 현업에서 많이 쓰던 방식이고 이 두가지를 합한 방식이 `GetMapping` 입니다.

## PathVariable

위처럼 우리가 주소를 지정하면 해당 URI 주소로 이동할 수 있게 됩니다. 하지만 매번 주소를 지정하기위해 GetMapping을 사용해 method를 만들 수 없기때문에 바뀌는 부분을 `PathVariable` annotation을 이용해 만드는 방법이 있습니다.

```java
		// http:// localhost:8080/api/get/path-variable/{name}
    @GetMapping("/path-variable/{name}")
    public String pathVariable(@PathVariable String name){
        System.out.println("PathVariable : "+name);
        return name;
    }
```

여기서 중요한 부분은 PathVarible 안의 변수 이름{name}과 String의 변수 이름이 같아야하는 것입니다. 하지만 만약 개발하는 도중에 이름이 달라져야할 때는 아래와 같이 설정해줌으로써 해결할 수 있습니다.

```java
		// http:// localhost:8080/api/get/path-variable/{name}
    @GetMapping("/path-variable/{name}")
    public String pathVariable(@PathVariable(name = "name") String pathName){
        System.out.println("PathVariable : "+ pathName);
        return pathName;
    }
```

## Query Parameter

Query Parameter는 검색을 할때 여러가지 매개변수 인자를 의미한다. 

예를들어 인텔리제이를 검색했을 때의 주소를 보게되면 아래와 같습니다.

[https://www.google.com/search?q=intellij&rlz=1C5CHFA_enKR959KR961&oq=intellij&aqs=chrome..69i57j0i512l2j0i67i131i433j0i512l2j0i67j0i512l3.557j0j7&sourceid=chrome&ie=UTF-8](https://www.google.com/search?q=intellij&rlz=1C5CHFA_enKR959KR961&oq=intellij&aqs=chrome..69i57j0i512l2j0i67i131i433j0i512l2j0i67j0i512l3.557j0j7&sourceid=chrome&ie=UTF-8)

여기서 ? 뒤의 모습을 보면 &연산자로 key와 value들을 받아오는 것을 볼 수가 있습니다. 이를 나눠보면 더 명확하게 보이게 됩니다.

- search?q = intellij
- &rlz = 1C5CHFA_enKR959KR961
- &oq = intellij&aqs=chrome..69i57j0i512l2j0i67i131i433j0i512l2j0i67j0i512l3.557j0j7
- &sourceid = chrome
- &ie = UTF-8

이렇게 key를 통해 검색하기 위해 `?key=value&key2=value2` 형식으로 입력하는 것을 `Query Parameter`라 합니다.

이를 스프링에서는 어떻게 받아오는지는 아래의 코드와 같습니다.

```java
		// http://localhost:8080/api/get/query-param?user=steve&email=steve@gamail.com&age=30
    @GetMapping(path = "query-param")
    public String queryParam(@RequestParam Map<String, String> queryParam){
        StringBuilder sb = new StringBuilder();

				//lamda (알아볼 것)
        queryParam.entrySet().forEach(entry -> {
            System.out.println(entry.getKey());
            System.out.println(entry.getValue());
            System.out.println("/n");

            sb.append(entry.getKey()+" = "+ entry.getValue());
        });

        return sb.toString();
    }
```

이렇게 Query Param을 `RequestParam` annotation으로 받아올 수 가 있습니다. 하지만 이렇게 Map으로 받아버리면 무슨 키가 몇개인지 알 수가 없습니다. 이럴 땐 `RequestParam` 을 각각의 변수에 지정해주고 받으면 어떤 키를 받아오는지 명확하게 알 수가 있게 됩니다. 

```java
@GetMapping(path = "query-param02")
    public String queryParam02(
            @RequestParam String user,
            @RequestParam String email,
            @RequestParam int age
    ){
        System.out.println(user);
        System.out.println(email);
        System.out.println(age);

        return user +" " + email + " "+ age;
    }
```

여기서 또 주의할 것은 주소에 명시되어있는 Key값과 `RequestParam` 으로 받아오는 변수명이 일치해야 오류가 뜨지 않습니다. 또한 변수타입에 맞춰서 입력이 되어야 오류가 발생하지 않습니다. 

하지만 이같은 오류를 해결하는 방법은 따로 있습니다. 바로 미리 정의해놓는 것입니다. 이는 현업에서 자주쓰이는 방식입니다. 객체를 만들어서 매개변수로 넘겨주는 방식이고 코드는 아래와 같습니다.

```java
package com.example.hello.dto;

public class UserRequest {

    private String name;
    private String email;
    private int age;

    public String getName() {
        return name;
    }

    public void setName(String name) {
        this.name = name;
    }

    public String getEmail() {
        return email;
    }

    public void setEmail(String email) {
        this.email = email;
    }

    public int getAge() {
        return age;
    }

    public void setAge(int age) {
        this.age = age;
    }

    @Override
    public String toString() {
        return "UserRequest{" +
                "name='" + name + '\'' +
                ", email='" + email + '\'' +
                ", age=" + age +
                '}';
    }
}
```

먼저 `UserRequest` 라는 객체를 만들고 나서 아래와 같이 매개변수로 넘겨줍니다. 이때 주의할 점은 `@RequestParam` annotation을 쓰지 않는다는 것입니다. 이는 스프링에서 객체를 생성해 받을때 자동으로 객체 안의 변수들과 매칭 시켜주기 때문입니다.

```java
		@GetMapping(path = "query-param03")
    public String queryParam03(UserRequest userRequest){
        System.out.println(userRequest.getName());
        System.out.println(userRequest.getEmail());
        System.out.println(userRequest.getAge());

        return userRequest.toString();
    }
```

**용어설명**

- `dto`
    
     Data Transfer Object의 약자로 계층간 데이터를 전송하기 위해 사용되는 객체
    
    로직은 없고 순수 데이터만 (getter & setter로만) 이루어진 객체이다.
