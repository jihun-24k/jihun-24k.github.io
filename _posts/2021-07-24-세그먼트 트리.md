### 세그먼트 트리 개념

- **선형구조 (ex 배열) 안의 값들을 구간별로 나누어 원하는 정보를 저장한 트리**
    - 대표적으로 **구간 합** 알고리즘이 있다
- **이진 트리로 구성**
    - 만일 N이 2의 제곱꼴이면 완전 이진 트리
- **세그먼트 트리의 구성**
    - 리프 노드: 배열의 그 수 그 자체
    - 리프 외 노드: 왼쪽 자식 노드와 오른쪽 자식 노드의 합

---

### 세그먼트 트리를 사용하는 이유

배열 A가 있고 아래와 같이 두 연산을 해야하는 문제에서 **수행해야하는 연산은 최대 M번**

1. 구간 l, r (l ≤ r)이 주어졌을 때, A[l] + A[l+1] + ... + A[r-1] + A[r]을 구해서 출력하기
2. i번째 수를 v로 바꾸기. A[i] = v

- **세그먼트 트리나 다른 방법을 사용하지 않고 문제를 풀 경우**
    - 1번 연산을 수행하는데 O(N), 2번 연산을 수행하는데 O(1)이 걸리게 됩니다. 총 시간 복잡도는 O(NM) + O(M) = O(NM)이 나오게 된다
    - 2번 연산이 없을 땐 합이 바뀌지가 않아 차례대로 합을 구해주면 된다
    - 하지만 2번 연산이 있다면 해당되는 합들을 다시 다 바꿔줘야한다
    - 따라서, M과 N이 매우 큰 경우에는 시간이 너무 오래걸리게 된다

### 세그먼트 트리 만들기
- 원하는 구간의 정보를 담을 수 있게 트리를 만든다
- 재귀 함수를 통해 왼쪽 구간과 오른쪽 구간을 호출한다.

    ```python
    # 세그먼트 트리 생성
    tree = [0] * 3000000

    def init(node, start, end):
        if start == end:
            tree[node] = nums[start]
            return tree[node]
        else:
            tree[node] =  init(node * 2, start, (start + end)//2) + init(node * 2 + 1, (start + end)//2 + 1, end)
            return tree[node]
    ```

### 구간 합 구하기

- start, end는 노드의 구간 정보 // left, right는 입력받은 구간
- 구간의 합을 찾을 때의 4가지 경우의 수
    1. `[left,right]`와 `[start,end]`가 겹치지 않는 경우
    2. `[left,right]`가 `[start,end]`를 완전히 포함하는 경우
    3. `[start,end]`가 `[left,right]`를 완전히 포함하는 경우
    4. `[left,right]`와 `[start,end]`가 겹쳐져 있는 경우 (1, 2, 3 제외한 나머지 경우)
- 1 번일 경우는 구해야하는 구간이 아니기 때문에 0을 반환 해준다.
- 2 번일 경우는 더이상 재귀를 할 필요가 없기에 해당 tree 노드 값을 반환 해준다.
- 3, 4번일 경우는 1, 2번의 경우를 찾기 위해 다시 2분할로 나누어 재귀를 해준다.

```python
# 구간 합 찾기
def find_sum(node, start, end, left, right):
		# 겹치지 않는 경우
    if left > end or right < start:
        return 0
		# 완전히 포함하는 경우
    elif left <= start and end <= right:
        return tree[node]
		# 이외의 경우
    else:
        m1 = find_sum(node * 2, start, (start + end)//2, left, right)
        m2 = find_sum(node * 2 + 1, (start + end)//2 +1, end, left, right)
        return m1 + m2
```

### 수 바꾸기

- 해당 바꿔야하는 인덱스를 포함한 노드의 구간들만 바꿔준다

```python
# 수 바꾸기
def update(node, start, end, index, diff):
    if index < start or index > end:
        return
    tree[node] = tree[node] + diff     
    if start != end:
        update(node * 2, start, (start + end)//2, index, diff)
        update(node * 2 + 1, (start + end)//2 + 1, end, index, diff)
```

### 세그먼트 트리 시간 복잡도

1. 구간 l, r (l ≤ r)이 주어졌을 때, A[l] + A[l+1] + ... + A[r-1] + A[r]을 구해서 출력하기
2. i번째 수를 v로 바꾸기. A[i] = v
- 1번 2번 연산을 수행하는 것은 **O(lgN)** 만에 수행할 수 있다.
