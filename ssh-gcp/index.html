SSH로 GCP 원격 서버 접속하기
Cloud
2024-08-04

클라우드에서 가상 머신(VM)을 만들면 그 원격지에 있는 서버에 접근해 사용해야 한다. 요즘은 예전에 주로 사용되던 텔넷(Telnet)을 대체해 SSH 통신방법을 이용한다. SSH가 무엇인지, SSH를 통해 원격 서버에 어떻게 접속이 가능한지 정리했다.

SSH란?

SSH는 Secure SHell의 줄임말로 원격지에 있는 컴퓨터를 안전하게 제어하기 위한 프로토콜 또는 그 프로토콜을 사용하는 응용 프로그램을 뜻한다. 기본적으로 22번 포트를 통해 통신한다. 암호화 방법으로 통신하기 때문에 이전 텔넷이 가지고 있던 보안 취약점을 해결했다.

SSH 클라이언트

리눅스와 Mac 같은 유닉스(Unix) 계열의 운영체제는 기본적으로 SSH 클라이언트가 설치되어 있기에 SSH를 그냥 사용할 수 있다. 윈도우 운영체제는 윈도우 10과 윈도우 서버 2019 이후 버전부터 Power Shell과 OpenSSH를 지원한다. 하지만 그 이전 버전은 SSH가 지원되지 않아 SSH 클라이언트를 따로 설치해야한다. 대표적으로 윈도우에서 설치해 사용하는 SSH 클라이언트는 아래와 같다.

  • PuTTY

  • xShell

SSH 서버

유닉스 계열의 운영체제는 OpenSSH를 가장 많이 사용된다. OpenSSH는 SSH 클라이언트와 서버가 포함되어있다. 대체로 유닉스 계열의 운영체제는 이미 설치가 되어있어 SSH를 위해 할 것은 없다. 윈도우 운영체제는 앞선 클라이언트와 마찬가지로 윈도우 10과 윈도우 서버 2019 이후 버전부터 OpenSSH가 기본으로 지원된다.

SSH로 GCP 원격 서버 접속 방법

SSH는 암호화 방법으로 클라이언트와 서버 간 통신한다. 따라서 서로가 안전한지 인증할 수 있어야 한다. 이때 인증을 키(Key)를 통해서 진행한다. GCP 원격 서버에 접속하기 위해 다양한 방법이 있다. 하지만 암호화 키를 생성해 GCP 원격 서버에 등록한 뒤 접속하는 과정은 동일하다.

Google Cloud 콘솔로 접속

VM 인스턴스 페이지에 접속해 SSH을 누르면 접속이 가능하다. GCP가 자동으로 키를 생성해 인증해주기 때문에 다른 설정은 필요 없다.

gcloud CLI로 접속

  1. 먼저 아래와 같이 상단의 Cloud Shell 버튼을 누른다.

    Cloud Shell은 이미 gcloud CLI이 설치되어 있어 바로 명령어 사용이 가능하다. 로컬 환경에서 접속하려면 gcloud CLI를 따로 설치한다.

  2. 아래 명령어를 실행시킨다. 각 옵션은 아래 조건에 맞게 작성한다. 

gcloud compute ssh --project=[프로젝트 ID] --zone=[영역] [인스턴스 이름]

  • 프로젝트 ID

    • 접속하고자 하는 인스턴스가 포함된 프로젝트의 ID

  • 영역

    • 접속하고자 하는 인스턴스가 속한 영역

  • 인스턴스 이름

    • 접속하고자 하는 인스턴스의 이름

프로젝트 ID와 영역은 환경변수로 설정하면 따로 작성할 필요없이 인스턴스 이름만으로도 접속이 가능하다. 나에게 맞는 조건을 작성한 예시는 다음과 같다.

gcloud compute ssh --project=steel-aria-343404 --zone=us-central1-a instance-1

Google Cloud에 명령어를 처음 실행시키면 SSH 키를 만들 것인지 질문한다. 앞선 콘솔과는 달리 SSH 키를 수동으로 만들어야 한다. SSH 키를 생성한다고 입력하고 비밀번호를 설정해주면 원격 서버와 연결된다. 비밀번호는 공란도 가능하다.

로컬에서 접속

ssh-keygen 프로그램을 이용해서 SSH 키를 만든다. 해당 로컬은 Mac, 즉 유닉스 계열에서 키를 생성하는 방법이다.

  1. SSH 키 생성하기

    ssh-keygen -t rsa -f ~/.ssh/[키 파일명] -C [사용자 이름]

    먼저 암호화 키를 생성한다. SSH 키를 암호화할 방법(rsa), 생성될 키 파일명, 인스턴스 접속시 사용되는 사용자 이름을 설정해준다. 이때 키 파일명을 지정해주지 않으면 다른 SSH 키가 중복될 수 있다.

    passphrase 를 입력한다. passphrase는 일종의 비밀번호로 비공개키를 입력한 값으로 암호화한다. 권장 값은 10~30 문자이고 생략 가능하다. 자동 로그인을 원한다면 생략해야 한다. ls -al ~/.ssh/ 명령어로 SSH 키가 잘 생성 됐는지 확인이 가능하다.

    각 파일의 권한을 잘 설정해야 한다.

  2. 생성된 공개키를 GCP 원격 서버에 등록

    VM 인스턴스 설정에서 SSH public key 등록

    앞선 공개키 파일의 내용을 복사해 인스턴스 세부정보 페이지에 추가한다.

  3. 터미널에서 원격 서버 접속

    ssh -i [비공개키 파일 경로] [사용자 이름]@[외부 IP 주소]

    명령어를 입력하면 ssh 접속이 되는 것을 확인할 수 있다.
