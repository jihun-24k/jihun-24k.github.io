HTTP 통신

웹프로그램은 대부분 HTTP 통신 통해 데이터를 주고 받는다. 이렇게 HTTP 통신을 하자고 하면 통신 규약이 있어야 하는데 그것을 프로토콜, 즉 HTTP 통신 프로토콜이라고 부른다. 여러 규약 중 요청(Request), 응답(Response) 데이터를 어떤 구조로 통신을 하는지 알아보자.

InputStream & OutputStream

먼저 HTTP 요청/응답 데이터 구조를 알아보기 전에 클라이언트와 서버 사이에 일어나는 통신은 어떤 데이터 형태로 주고 받는지 알면 더 좋을 것 같아 먼저 짚고 넘어가고자 한다. 웹서버는 브라우저에서 오는 데이터를 Stream으로 변환해서 주고 받는다. 이때 브라우저에서 요청하는 데이터는 InputStream 형태로 받고 내보낼때는 OutputStream 형태로 변환해서 처리한다.

HTTP 요청 데이터 구조

먼저 InputStream으로 받는 HTTP 요청 데이터는 위의 그림과 같이 구성되어있다.

  • 요청 라인

    • 요청 라인은 HTTP 메서드, 경로(url), http 버전으로 구성 되어있다.

  • 요청 헤더

    • 요청에 대한 추가 정보들을 담는 공간이다.

  • 빈 공백

    • 헤더와 본문을 구분하기 위한 공백열이다.

  • 요청 본문

    • POST 또는 PUT 요청과 함께 전송되는 데이터이며 선택적이다.

HTTP 응답 데이터 구조

OutputStream으로 내보내는 HTTP 응답 데이터도 요청 데이터와 마찬가지인 구조를 갖지만 응답 라인의 구조가 살짝 다르다.

  • 응답 라인

    • 응답 라인은 http 버전, 상태코드, 상태 메시지로 구성 되어있다.

  • 응답 헤더

    • 응답에 대한 추가 정보를 담는 공간이다.

  • 빈 공백

    • 헤더와 본문을 구분하기 위한 공백열이다.

  • 응답 본문

    • 응답으로 전송되는 데이터이다.

    • 여기 예시로는 Hello World가 표시되는 html 파일을 담아서 보내준다.
