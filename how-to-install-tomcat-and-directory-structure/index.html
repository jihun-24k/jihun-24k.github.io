톰캣 설치 및 기본 디렉토리 구조
WEB
2024-09-16

서버에 웹 애플리케이션을 톰캣을 이용해 배포하고자 한다. 톰캣의 기본 구조와 설치, 설치 후 설정에 대해 간략하게 정리했다.

톰캣 설치

톰캣 설치 환경

  • 윈도우는 zip, 리눅스는 tar 파일로 다운 받는다.

  • 톰캣의 버전과 JDK의 버전이 호환되는지 확인한다.

    • 예를 들어 Tomcat 9.0.x, 10.0.x, 8.5.x: JDK 11과 호환된다.

    • 하지만 Tomcat 8.0.x 및 그 이전 버전: JDK 11과 호환되지 않으며, JDK 8 이하에서만 사용 가능하다.

톰캣 설치 과정

리눅스에서 설치 방법에는 두 가지 방법이 있다.

  1. wget으로 압축된 파일(tar.gz)을 받아와 풀기

    wget 명령어를 사용해서 톰캣을 다운 받는다. 예를 들어 Tomcat 9.0.70 버전을 다운로드하려면 다음과 같이 입력한다.

    wget <https://downloads.apache.org/tomcat/tomcat-9/v9.0.70/bin/apache-tomcat-9.0.70.tar.gz>

    tar 파일 압축을 풀면 톰캣 설치가 완료된다.

    tar -xvzf apache-tomcat-9.0.70.tar.gz

  2. 리눅스가 인터넷 연결이 되어있지 않으면 ftp로 받아와야함

톰캣 설치 후 작업

JAVA_HOME 설정

JAVA_HOME을 설정하는 방법은 두 가지가 있다.

  • 환경 변수 PATH로 java home을 설정하는 방법

  • 각 스크립트에 넣어서 설정하는 방법

PATH와 script의 차이점

  • 환경변수 PATH는 모든 자바 리소스가 동일한 자바 버전으로 동작하게 된다.

  • script는 각 리소스당 자바 버전을 설정해서 동작하게 한다.
